Gemini CLI 명령어 가이드

Gemini CLI에서 사용할 수 있는 주요 슬래시 명령어(/) 일람입니다.

일반 및 도움말
명령어 설명
/about현재 CLI 버전 및 환경 정보를 표시합니다.
/help (또는 /?)사용 가능한 명령어 목록과 기본 사용법을 보여줍니다.
/docs브라우저에서 Gemini CLI 문서를 엽니다.
/privacy개인정보 보호 고지 및 데이터 수집 설정을 표시합니다.
/upgradeGemini Code Assist 업그레이드 페이지를 엽니다.
/bug <headline>제공된 제목으로 GitHub에 이슈를 제보합니다.
세션 및 컨텍스트 관리
명령어 설명
/chat / /resume대화 체크포인트 및 세션 브라우징을 관리합니다. (list, save, share 등)
/rewind히스토리를 뒤로 돌려 대화 상태나 파일 변경 사항을 되돌립니다.
/restore [id]특정 도구 실행 전의 상태로 파일을 복구합니다.
/clear터미널 화면과 세션 히스토리를 지웁니다.
/compress대화 컨텍스트를 요약으로 대체하여 토큰 사용량을 최적화합니다.
/quit (또는 /exit)CLI를 종료합니다. --delete 옵션으로 히스토리 삭제가 가능합니다.
도구, 확장 및 에이전트
명령어 설명
/tools사용 가능한 도구 목록을 나열합니다. desc를 붙이면 상세 설명을 봅니다.
/extensionsCLI 확장 기능을 관리합니다 (설치, 업데이트, 목록, 활성화 등).
/mcpModel Context Protocol(MCP) 서버를 관리합니다.
/skills특수 워크플로우를 위한 에이전트 기술(Skills)을 관리합니다.
/agents멀티 에이전트 작업을 위한 로컬 및 원격 서브 에이전트를 관리합니다.
/commands.toml 파일에서 로드된 사용자 정의 명령어를 관리합니다.
/hooksCLI 동작을 가로채고 커스터마이징하는 라이프사이클 훅을 관리합니다.
설정 및 UI
명령어 설명
/settings대화형 설정 에디터를 엽니다.
/themeCLI의 시각적 테마를 변경하는 대화창을 엽니다.
/model활성 AI 모델을 설정합니다 (예: set <model-name>).
/editor파일 작업에 사용할 기본 에디터를 선택합니다.
/vim입력 영역에서 Vim 스타일의 탐색 및 편집 모드를 토글합니다.
/terminal-setup여러 줄 입력을 위한 터미널 키바인딩을 설정합니다.
프로젝트 및 워크스페이스 제어
명령어 설명
/directory (또는 /dir)다중 디렉토리 워크스페이스를 관리합니다 (디렉토리 추가/표시).
/memoryGEMINI.md 파일의 계층적 지침 컨텍스트를 관리합니다.
/init프로젝트별 GEMINI.md 컨텍스트 파일을 생성합니다.
/permissions폴더 신뢰 설정 및 권한을 관리합니다.
/plan계획 모드(읽기 전용)로 전환하여 생성된 작업 계획을 검토합니다.
/policies활성 보안 및 실행 정책 목록을 보여줍니다.
유틸리티
명령어 설명
/stats세션, 모델(토큰 수), 도구 사용 통계를 상세히 표시합니다.
/shells (또는 /bashes)백그라운드 쉘 프로세스 뷰를 토글합니다.
/copy마지막 AI 응답을 시스템 클립보드에 복사합니다.
/ideIDE 통합 및 컴패니언 설치를 관리합니다.
/setup-githubPR 및 이슈 자동 처리를 위한 GitHub Actions를 설정합니다.
